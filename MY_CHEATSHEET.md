## Python: типы данных

### Мутабельные (изменяемые)
- списки (`list`)

## Python: тонкости, которые я просил запомнить

| Тема | Что запомнить |
|------|---------------|
| `split()` | `split()` — разбивает по любому пробелу, игнорирует пустые строки. `split(" ")` — разбивает по каждому пробелу, сохраняет пустые |
| `extend()` | Добавляет элементы из итерируемого объекта (не одиночный объект) |
| `isinstance()` | Учитывает наследование. Лучше `type()` |
| `issubclass()` | Проверяет, является ли класс наследником |
| `popitem()` | Удаляет и возвращает последнюю пару (ключ, значение) из словаря |
| `zip()` | "Сшивает" последовательности в кортежи |
| `get()` vs `setdefault()` | `get` — безопасный поиск. `setdefault` — как get, но если нет ключа — создаёт |
| `del` | Удаляет ключ (опасно — KeyError) |
| `@functools.wraps` | Сохраняет `__name__` и `__doc__` функции в декораторе |
| `frozenset` | Неизменяемое множество. Нужен для ключей словаря из kwargs |
| `nonlocal` | Нужен, чтобы менять переменную из замыкания |
| `perf_counter()` | Точнее, чем `time()` для измерения производительности |
| `*args` | Кортеж позиционных аргументов |
| `**kwargs` | Словарь именованных аргументов |
| `yield from` | Делегирование генератора |
| `next(gen, default)` | Безопасное получение следующего элемента |
| `csv` модуль | `csv.reader` → списки, `csv.DictReader` → словари |
| `hasattr()` | Проверяет наличие атрибута |
| `__iter__` | Магический метод, делает объект итерируемым |

## Python: память и GIL

### Интернирование (interning)
- Малые целые числа от **-5 до 256** кэшируются
- `a is b` может быть `True` для этих чисел, даже если присваивали отдельно

### Области памяти
| Что | Где хранится |
|-----|--------------|
| Ссылки на локальные переменные | Стек |
| Все объекты (списки, словари) | Куча |
| Глобальные переменные | Сегмент данных |
| Байт-код функций | Сегмент кода |

### GIL (Global Interpreter Lock)
- Глобальная блокировка интерпретатора
- Не позволяет выполнять больше одного потока Python-кода одновременно
- **CPU-задачи** (вычисления) — многопоточность НЕ ускоряет
- **I/O-задачи** (ожидание диска/сети) — многопоточность ускоряет
- Для параллельных вычислений — `multiprocessing`

## Git (основные команды)

```bash
git init                    # сделать папку репозиторием
git add .                   # добавить все файлы на "стол"
git commit -m "сообщение"   # сохранить коммит
git push origin main        # отправить на GitHub
git pull origin main        # скачать изменения с GitHub
git remote -v               # посмотреть удаленные репозитории
git log --oneline           # краткая история коммитов
git status                  # что изменилось с последнего коммита
