## Python: память и GIL

### Интернирование (interning)
- Малые целые числа от **-5 до 256** кэшируются
- `a is b` может быть `True` для этих чисел, даже если присваивали отдельно
- `is` сравнивает адреса в памяти. ОН не заглядывает в кучу, а просто проверяет, один и тот же это объект или разные.
- `==` сравнивает значения объектов


### Области памяти
| Что | Где хранится |
|-----|--------------|
| Ссылки на локальные переменные | Стек |
| Все объекты (списки, словари) | Куча |
| Глобальные переменные | Сегмент данных |
| Байт-код функций | Сегмент кода |

### GIL (Global Interpreter Lock)
- Глобальная блокировка интерпретатора
- Не позволяет выполнять больше одного потока Python-кода одновременно
- **CPU-задачи** (вычисления) — многопоточность НЕ ускоряет
- **I/O-задачи** (ожидание диска/сети) — многопоточность ускоряет
- Для параллельных вычислений — `multiprocessing`
#### Точнее
- GIL не разрешает нескольким потокам (threads) выполняться одновременно в рамках одного процесса. Процессы (multiprocessing) — другое дело, у каждого свой GIL
- Многопоточность ускоряет I/O-задачи (ожидание диска, сети, базы данных). Для CPU-задач (вычисления) многопоточность не ускоряет, а даже замедляет из-за переключения контекста
- Параллельно (одновременно на разных ядрах) потоки не выполняются из-за GIL. Они выполняются конкуренто (переключаются очень быстро, создавая иллюзию параллельности)
- CPU-bound — вычисления в процессоре. I/O-задачи — это когда процессор простаивает в ожидании (диск, сеть, пользователь)

## Git (основные команды)

```bash
git init                    # сделать папку репозиторием
git add .                   # добавить все файлы на "стол"
git commit -m "сообщение"   # сохранить коммит
git push origin main        # отправить на GitHub
git remote -v               # посмотреть удаленные репозитории
git pull origin main        # скачать изменения с GitHub
git remote -v               # посмотреть удаленные репозитории
git log --oneline           # краткая история коммитов
git status                  # что изменилось с последнего коммита
```

## Git: конфликты и их разрешение

### Что такое конфликт
Git не может сам решить, какую версию файла оставить, потому что изменения перекрываются.

### Почему возникает
- Два человека изменили одну и ту же строку в одном файле
- Или: ты изменил файл локально, а на GitHub уже есть другие изменения (забыл сделать `git pull`)

### Как разрешить
1. Открыть конфликтный файл (Git добавит маркеры)
2. Оставить нужные строки, убрать маркеры `<<<<<<<`, `=======`, `>>>>>>>`
3. `git add . && git commit -m "Разрешил конфликт"` # отдельная точка '.' внутри команды означает "Текущая папка и всё, что внутри"
4. `git push origin main`

### Как избежать
Всегда перед редактированием: `git pull origin main`

## Git: полезные сокращения и операторы

### Точка (.)
- `git add .` — добавить ВСЕ измененные файлы
- `cd .` — остаться в текущей папке
- `./script.py` — запустить скрипт из текущей папки

### Операторы в терминале
- `&&` — выполнить вторую команду, если первая успешна
- `||` — выполнить вторую, если первая провалилась
- `;` — выполнить вторую в любом случае

## SQL: агрегация и группировка

SELECT в SQL не выводит на экран. Он возвращает таблицу (набор строк)
### Агрегатные функции
- `COUNT(*)` — количество строк - считает все строки
- `COUNT(col)` — считает строки, где значение в столбце col не NULL
- `SUM(col)` — сумма
- `AVG(col)` — среднее
- `MIN(col)` — минимум
- `MAX(col)` — максимум
### Математика
- `ABS(column)` — модуль числа
- `+-*/` — определены только для чисел (целых и дробных)
- `/` — выводит дробное число, если либо делимое, либо делитель тоже дробное (можно обозначить так: "число::numeric"), иначе целая часть без остатка
### Нумерация строк
|Функция | Что делает |	Пример (при одинаковых баллах)|
|--------|------------|-------------------------------|
|ROW_NUMBER() |	Присваивает уникальный номер каждой строке |	1, 2, 3, 4...|
|RANK() |	Одинаковый балл → одинаковый ранг, следующий пропускает |	1, 2, 2, 4...|
|DENSE_RANK()|	Одинаковый балл → одинаковый ранг, следующий не пропускает |	1, 2, 2, 3...|

### `AS` — алиас (временное имя)
```sql
SELECT AVG(salary) AS avg_salary FROM employees;
AS можно не писать, но с AS читаемее
```
## Фильтрация, группировка и сортировка
`GROUP BY column` — Группировка по столбцу column. Агрегатные функции (`COUNT`, `SUM`, `AVG`) применяются к каждой группе отдельно.
`<функция> OVER (PARTITION BY column)` — Добавляем новый столбец, в котором будет значение функции для всей группы (способ группировки)
`OVER ()` без `PARTITION BY` — окно = вся таблица.
```sql
SELECT 
    name,
    score,
    AVG(score) OVER () AS total_avg
FROM applicants;
```
`ORDER BY column DESC/ASC` — Сортировка по столбцу column по УБЫВАНИЮ/ВОЗРАСТАНИЮ
`WHERE` — Фильтрация каждой строки по значению в столбце ДО группировки
`HAVING` — Фильтрация ПОСЛЕ группировки (и только после GROUP BY)

### csv-файлы
csv.reader — читает строки как списки:
```bash
import csv
with open('data.csv', 'r') as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)  # ['name', 'age', 'city']
                    # ['Алексей', '18', 'Москва']
```

csv.DictReader — читает строки как словари (первая строка — ключи):
```bash
import csv
with open('data.csv', 'r') as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row['name'], row['age'])  # Алексей 18
```
