## 1. Фундамент (Python, CS, Git)

### 1.1 Python: типы данных и структуры
- Мутабельность / иммутабельность (`id()`, `is` vs `==`)
- Строки: методы, срезы, f-строки, форматирование
- Списки: методы, срезы, list comprehensions
- Словари: методы, dict comprehensions, `get`, `setdefault`, `pop`, `popitem`
- Кортежи, множества, `frozenset`

### 1.2 Python: функции и декораторы
- Аргументы: позиционные, именованные, `*args`, `**kwargs`, значения по умолчанию (ловушка с мутабельными)
- Области видимости: `global`, `nonlocal`
- lambda-функции, замыкания (closures)
- Декораторы: базовые, с параметрами, композиция декораторов, `@functools.wraps`

### 1.3 Python: итераторы, генераторы, исключения
- Итераторы vs итерируемые объекты, протокол итератора (`__iter__`, `__next__`)
- Генераторы: `yield`, `yield from`, `next(gen, default)`
- Генераторные выражения (круглые скобки) vs list comprehensions (квадратные)
- Исключения: `try/except/else/finally`, `raise`, `assert`, типы исключений

### 1.4 CS фундамент (глубоко)
- **GIL (Global Interpreter Lock)**: CPU-bound vs I/O-bound, многопоточность (`threading`) vs многопроцессность (`multiprocessing`)
- **Память**: стек (ссылки, вызовы), куча (объекты), сегмент данных (глобальные переменные), сегмент кода (байт-код)
- **Управление памятью**: счетчик ссылок (reference counting), сборщик мусора (generational GC), циклические ссылки, `gc.collect()`
- **Интернирование (interning)**: малые целые числа (-5 до 256), короткие строки
- **Байт-код**: `dis`, инструкции (`LOAD_FAST`, `LOAD_CONST`, `BINARY_ADD`, `RETURN_VALUE`)
- **`is` vs `==`**: сравнение адресов vs сравнение значений
- **`sys.modules`**: кеш загруженных модулей, принудительная перезагрузка

### 1.5 Git (система контроля версий)
- **Основы**: репозиторий, коммит, ветка, слияние, удаленный репозиторий (`origin`)
- **Команды**: `init`, `add`, `commit`, `push`, `pull`, `clone`, `status`, `log`, `diff`
- **Ветки**: `branch`, `checkout`, `merge`, `branch -d`
- **Конфликты**: маркеры (`<<<<<<<`, `=======`, `>>>>>>>`), ручное разрешение, `merge --abort`
- **Золотое правило**: `pull → edit → add → commit → push`
- **`.gitignore`**: какие файлы не отслеживать

### 1.6 Инструменты (введение)
- Терминал (bash, cmd): основные команды (`ls`, `cd`, `pwd`, `echo`, `&&`, `||`, `;`)
- Виртуальные окружения (`venv`), `pip`, `requirements.txt`
- CSV формат: `csv.reader`, `csv.DictReader`

## 3. SQL: база, JOIN, оконные функции

### 3.1 Базовый синтаксис
- `SELECT`, `FROM`, `WHERE`, `ORDER BY`, `LIMIT`
- `NULL`: проверка через `IS NULL` (не `= NULL`)
- Точка с запятой `;` — конец запроса (в терминале БД)
- Строки — только в одинарных кавычках `'`
- Регистр не важен, но команды пишут ЗАГЛАВНЫМИ (стиль)

### 3.2 JOIN
- `INNER JOIN`: только строки с совпадением в обеих таблицах
- `LEFT JOIN`: все строки из левой таблицы + совпадения из правой (где нет — `NULL`)
- `RIGHT JOIN`: все строки из правой таблицы
- `FULL OUTER JOIN`: все строки из обеих таблиц
- `ON` vs `WHERE`: `ON` — условие соединения, `WHERE` — фильтрация после соединения

### 3.3 Оконные функции (текущая тема)
- `ROW_NUMBER()` — нумерация строк в рамках раздела
- `RANK()` и `DENSE_RANK()` — ранжирование с пропусками и без
- `LAG()` и `LEAD()` — доступ к предыдущим/следующим строкам
- `PARTITION BY` — разделение на группы внутри окна
- `OVER()` — определение окна
